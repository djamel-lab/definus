# Dashboard utilisateur

## Définition
Page d'accueil après connexion qui résume les informations importantes pour l'utilisateur connecté.

## Sous-features
- [ ] Résumé chiffres personnels (solde, projets, activité récente)
- [ ] Actions en attente (à approuver, à compléter)
- [ ] Dernières notifications
- [ ] Raccourcis vers les actions fréquentes
- [ ] Graphique d'évolution personnelle
- [ ] Personnalisation du dashboard (widgets déplaçables)

## Questionnaire de cadrage
1. Quelles informations afficher en priorité pour l'utilisateur connecté ?
2. Y a-t-il des actions en attente à mettre en avant ? (ex: dossier à compléter, document à signer)
3. Faut-il afficher les dernières notifications ?
4. Faut-il des raccourcis vers les actions fréquentes ?
5. Faut-il un graphique d'évolution ? (ex: performance du portfolio)
6. Le dashboard est-il le même pour tous les rôles ou différent ?
7. L'utilisateur peut-il personnaliser son dashboard ? (ajouter/supprimer des widgets)
