# Monitoring

## Définition
Surveillance de la santé technique de l'application et des workflows : uptime, erreurs, performance, exécutions.

## Sous-features
- [ ] Uptime monitoring
- [ ] Error tracking (Sentry, LogRocket)
- [ ] Performance monitoring (temps de réponse, Core Web Vitals)
- [ ] Dashboard d'exécution des workflows
- [ ] Alertes en cas de panne/erreur
- [ ] Logs centralisés

## Questionnaire de cadrage
1. Que faut-il monitorer ? (uptime, erreurs, performance, exécutions de workflows)
2. Quel outil ? (Sentry, Datadog, LogRocket, Grafana, Better Uptime)
3. Qui reçoit les alertes ? Par quel canal ? (email, Slack, SMS)
4. Faut-il un dashboard de monitoring accessible à l'équipe ?
5. Quels SLA ? (uptime 99.9%, temps de réponse < 200ms)
6. Faut-il des logs centralisés ? Quelle rétention ?
