# Tests automatisés

## Définition
Suite de tests qui vérifient automatiquement que l'application fonctionne correctement.

## Sous-types
- [ ] Tests unitaires (fonctions isolées)
- [ ] Tests d'intégration (modules combinés)
- [ ] Tests end-to-end (parcours utilisateur complet)
- [ ] Tests de snapshot (UI)
- [ ] Tests de charge / performance
- [ ] Tests de sécurité (OWASP)

## Questionnaire de cadrage
1. Quels types de tests ? (unit, integration, e2e)
2. Quel framework de test ? (Vitest, Jest, Playwright, Cypress)
3. Quel objectif de couverture de code ? (80%, 90%)
4. Les tests tournent-ils en CI à chaque PR ?
5. Faut-il des tests e2e ? Sur quels parcours ?
6. Faut-il des tests de performance / charge ?
7. Faut-il des tests de sécurité ? (OWASP top 10)
