# Accessibilité

## Définition
Rendre l'application utilisable par tous, y compris les personnes en situation de handicap. Conformité RGAA / WCAG.

## Sous-features
- [ ] Contraste suffisant (WCAG AA : 4.5:1 texte, 3:1 éléments UI)
- [ ] Navigation au clavier complète
- [ ] Attributs ARIA (aria-label, aria-describedby, role)
- [ ] Focus visible
- [ ] Alt text sur toutes les images
- [ ] Skip to content link
- [ ] Lecteur d'écran compatible
- [ ] Taille de texte ajustable (rem/em, pas de px)
- [ ] Animations désactivables (prefers-reduced-motion)
- [ ] Formulaires accessibles (labels, erreurs liées)

## Questionnaire de cadrage
1. Quel niveau de conformité ? (WCAG AA, WCAG AAA, RGAA)
2. Faut-il un audit d'accessibilité avant mise en ligne ?
3. L'application doit-elle être entièrement navigable au clavier ?
4. Faut-il tester avec un lecteur d'écran ? (VoiceOver, NVDA)
5. Y a-t-il des obligations légales d'accessibilité ? (service public, grande entreprise)
6. Les animations respectent-elles prefers-reduced-motion ?
