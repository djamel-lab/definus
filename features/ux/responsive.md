# Responsive design

## Définition
L'interface s'adapte à toutes les tailles d'écran : mobile, tablette, desktop.

## Sous-features
- [ ] Mobile-first design
- [ ] Breakpoints définis (mobile, tablet, desktop)
- [ ] Navigation adaptative (burger menu sur mobile)
- [ ] Images responsive (srcset, lazy loading)
- [ ] Tableaux responsive (scroll horizontal ou reformatage)
- [ ] Touch-friendly (cibles tactiles > 44px)
- [ ] Test sur devices réels

## Questionnaire de cadrage
1. Quels breakpoints ? (mobile 375px, tablet 768px, desktop 1280px, large 1440px)
2. L'approche est-elle mobile-first ou desktop-first ?
3. Y a-t-il des éléments qui disparaissent sur mobile ? (sidebar, colonnes)
4. Les tableaux de données sont-ils reformatés sur mobile ou scrollables ?
5. Faut-il tester sur des devices spécifiques ?
6. Y a-t-il des contraintes d'impression ? (mode print CSS)
