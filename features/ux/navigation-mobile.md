# Navigation mobile

## Définition
Patterns de navigation spécifiques aux apps mobiles.

## Sous-features
- [ ] Tab bar (bottom navigation, 3-5 onglets)
- [ ] Drawer (menu latéral swipeable)
- [ ] Stack navigation (push/pop de pages)
- [ ] Header avec back button
- [ ] Gestes (swipe back, swipe to delete)
- [ ] Haptic feedback
- [ ] Safe area (notch, barre de navigation)

## Questionnaire de cadrage
1. Quel pattern principal ? (tab bar, drawer, stack)
2. Combien d'onglets dans la tab bar ? (3-5 max) Lesquels ?
3. Faut-il un drawer secondaire ?
4. Faut-il des gestes ? (swipe back, swipe to delete, pull to refresh)
5. Faut-il du haptic feedback sur certaines actions ?
